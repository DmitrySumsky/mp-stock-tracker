## Что это за проект

**MP Stock Tracker** — Google Apps Script, который работает внутри Google Sheets.
Цель проекта — полная автоматизация операционных и финансовых процессов бизнеса на маркетплейсах. Проект активно развивается: текущие модули — только начало, новые будут добавляться по мере роста задач.

Текущие модули (v3.0):

1. **Остатки маркетплейсов** — собирает остатки товаров с OZON и WB по всем кабинетам ИП через их API, записывает в отдельные листы, уведомляет в Telegram.
2. **Остатки МойСклад** — выгружает остатки со складов с себестоимостью и ценой продажи в лист «Остатки МС».
3. **Кредиты** — синхронизирует остатки задолженностей по кредитам из листа «Кредиты Import» в управленческий баланс.

Архитектура намеренно модульная — каждый новый модуль добавляется как отдельный файл, не затрагивая существующий код.

Владелец: **DmitrySumsky** (dmitriy.sumski@gmail.com).

### Как GAS выполняет файлы

Google Apps Script загружает все `.gs` файлы в **одном глобальном скоупе** — переменные и функции из одного файла видны во всех остальных. Нумерация `00_`…`99_` задаёт порядок загрузки (важно: константы из `00_constants.gs` должны быть доступны всем остальным).

### Центральный конфиг — лист «Панель управления»

Все токены и настройки хранятся **только в таблице**, не в коде:

| Ячейка | Содержимое |
|--------|------------|
| B2 | Telegram Bot Token |
| B3 | Chat ID(s), через `;` |
| B5 | API-токен МойСклад |
| Строки 9+ | Кабинеты маркетплейсов (MP, ИП, ключи, имя листа) |

Функции чтения: `loadTelegramConfig()`, `getMsToken_()`, `loadCabinets()` — в `02_config.gs`.

## Правила работы над проектом

### 1. Никаких токенов и credentials в коде

Все секреты читаются из таблицы через функции из `02_config.gs`. Если нужен новый токен — добавляй поле в `initPanel()` (`01_panel.gs`) и читалку в `02_config.gs`.

### 2. Один модуль — один файл

Новый независимый модуль = новый файл `NN_name.gs` в `src/`. Нумерация по порядку. Не добавляй функции нового модуля в существующие файлы.

### 3. Константы только в `00_constants.gs`

Все `const` верхнего уровня — только в `00_constants.gs`. В остальных файлах локальные переменные внутри функций — ок.

### 4. Приватные функции с суффиксом `_`

Вспомогательные функции, не предназначенные для прямого вызова из меню — именуются с `_` на конце: `getMsToken_()`, `fetchMsPage_()`. Публичные (вызываемые из меню или триггеров) — без `_`.

### 5. Меню только в `99_menu.gs`

Все пункты меню добавляются исключительно в `onOpen()` в файле `99_menu.gs`. Структура: главное меню `Автоматизация` → три основных подменю (Остатки маркеты / Остатки МС / Кредиты) + Сервис.

### 6. Логирование через `writeLog()`

Все операции с данными логируются через `writeLog(mp, ip, status, details)` из `12_utils.gs`. Лог пишется в лист «Лог».

### 7. HTTP через `fetchWithRetry()`

Все внешние запросы — через `fetchWithRetry()` из `12_utils.gs` (3 попытки, экспоненциальный backoff). Прямые вызовы `UrlFetchApp.fetch()` только в healthCheck и там, где retry не нужен.

## Деплой и рабочий цикл

### Инструменты

- **clasp** — синхронизирует `src/*.gs` с Google Apps Script
- **git + GitHub** — версионирование (`DmitrySumsky/mp-stock-tracker`)
- `.clasp.json` — не в git, содержит `scriptId` конкретной таблицы

### Команды деплоя

```bash
# После изменений — коммит в git и пуш в таблицу:
cd "c:/Users/Lenovo/Desktop/cs education/mp-stock-tracker"

git add .
git commit -m "описание изменений"
git push                  # → GitHub (история версий)
clasp push --force        # → Google Таблица (актуальная версия)
```

### Тестовая и основная таблица

Сейчас `.clasp.json` указывает на **тестовую копию**. Когда всё проверено — меняем `scriptId` на основную таблицу и делаем `clasp push --force`.

## Добавление нового модуля (чеклист)

1. Создать `src/NN_name.gs`
2. Если нужны константы — добавить в `00_constants.gs`
3. Если нужно поле в панели — добавить в `initPanel()` (`01_panel.gs`) и читалку в `02_config.gs`
4. Добавить пункт(ы) меню в `onOpen()` (`99_menu.gs`)
5. Обновить `CHANGELOG.md`
6. `git commit` + `git push` + `clasp push --force`

## Известные ограничения GAS

- Максимальное время выполнения скрипта: **6 минут**. При большом числе кабинетов могут потребоваться батчи.
- `UrlFetchApp` делает запросы от имени Google — некоторые API это учитывают.
- Триггеры хранятся в Google-аккаунте пользователя, не в скрипте. При `removeTriggers()` удаляются ВСЕ триггеры проекта.
- GAS не поддерживает `import/export` — все файлы в одном глобальном скоупе.
